# /edu-diagnosis 学力诊断

> 分流：这里管学习/认知/技能类的卡点（数数、分类、语言、自理动作等）。孩子发脾气、打人、黏人、不肯睡这类情绪和社会性行为问题，去 `/edu-behavior`。

## 做法

1. 让用户描述（或你设计一个）具体任务场景与孩子的表现。
2. 从发展/学习维度拆解：他卡在"数数"还是"一一对应"？是"不会比大小"还是"不懂数量保留"？是"不会轮流"还是"不会自己收"？
3. 区分"发展顺序上的正常阶段"与"需要关注的信号"。
4. 产出：观察记录（卡点在哪、属于哪个发展维度）+ 2-3 个下一阶段的小活动建议。

## 输出格式

- **观察**：孩子在____任务里，表现为____。
- **卡点研判**：属于【____维度】，这是发展顺序中的____阶段 / 需关注信号是____。
- **下一步**：活动1（一句话）/ 活动2 / 活动3。
- ［非定论标识：观察记录，非测评结论；重大安排请咨询早教/幼教专业人士。］

## 护栏

- 只做观察记录，不做"智商/能力/落后"判定。
- 出现"这孩子不行"类表述一律改写为发展描述。
- 依据引《3-6 岁儿童学习与发展指南》及各发展领域共识，不凭空判断。

## 案例（3 正面 + 2 反面）

**正面**
1. 一个 4 岁男孩数积木，总漏掉最后一块就说"数完了"。skill 先记"点数时一一对应还不稳"，再给下一步建议：让他数完指着一个说"这是最后一个，一共几块"，而不是说"你数学不行"。
2. 一个 5 岁女孩分碗筷，给每人放一个后多问一句"每人都有吗"。skill 记下"已能一一对应，正往'自查总数'走"，建议下一步玩"拉开一排再比一样多"的守恒铺垫。
3. 一个 3 岁孩子比两堆积木，说不清谁多，只盯着高的那堆。skill 记"这是中心化阶段，一次只盯一个特征，属正常发展顺序"，建议用并排配对比较，不急着教"一样多"。

**反面**
1. 一个 3 岁孩子还数不到 10，家长就说"他数学差、得报班"。skill 若照这个下结论就走偏了：3 岁本就处在机械唱数期，应先观察是"不会数"还是"正常阶段"，别把发展节奏当能力缺陷。
2. 一位老师想看 4 岁孩子"懂不懂加减"，直接发了一张纸笔算式卷。这偏离了观察的本意：幼儿该在分饼干、搭积木的真实任务里看表现，用试卷既测不出真理解，还容易给孩子贴"不会"的标签。
