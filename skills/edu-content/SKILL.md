# /edu-content 内容诊断

## 做法

1. 用户给一个内容（活动方案 / 绘本 / 视频 / 教具）。
2. 从"是否触发思维"判：是练记忆、练操作、还是练思维（比较 / 分类 / 推理 / 表征）？
3. 指出哪步在"假启蒙"（看似数学实则死记）。
4. 产出：诊断结论 + 改造清单（怎么改成真启蒙）。

## 输出格式

- **判定**：该内容主要练的是____（记忆/操作/思维中的____）。
- **假启蒙点**：____（如只背数词不建立数感，或只背古诗不理解意思）。
- **改造**：改法1 / 改法2（各一句话）。
- ［非定论标识：观察记录，非测评结论；重大安排请咨询早教/幼教专业人士。］

## 护栏

- 评价的是"内容设计"，不评价使用它的孩子。
- 不把"热闹/孩子喜欢"等同于"发展了思维"。
- 改造建议依据发展心理学与《指南》，不凭流行说法。

## 案例（3 正面 + 2 反面）

**正面**
1. 一个号称"数学启蒙"的 APP 只要孩子跟着念 1 到 100。skill 判定它主要练记忆，假启蒙点是"背数词不等于懂数"，改造建议：加一步让孩子手口一致点数实物、数完说"一共几个"，把数词和真实数量绑起来。
2. 一本绘本被设计成"这页讲了什么、作者是谁"的连番考问。skill 指出这是把共读变考核、练的是回忆而非思维，改造建议：改成对话式，多问"他现在心里在想什么""你有没有过一样的时候"，让情绪和推理被练到。
3. 一套"按高矮排顺序"的卡片活动，让孩子排完说"谁最高、谁最矮、中间是谁"。skill 判定它真在练比较与排序思维，正面肯定，并建议加一句"如果再来一个更矮的放哪"延伸推理。

**反面**
1. 一个视频宣传"10 天学会 100 个字"，全程是字卡闪卡跟读。skill 若把它当启蒙就错了：脱离语境机械识字练的是记忆，且超前过量易透支兴趣，改造应回到"在故事和生活中自然认字"，不是堆字表。
2. 一个热闹的古诗吟唱视频，孩子跟着又唱又跳很开心，就被宣传成"开发了思维"。这踩了护栏：把"孩子喜欢、场面热闹"等同于"发展了思维"是误判，诗的意思没被理解就只是娱乐和记忆，诊断应点出"懂意"这一步缺失。
