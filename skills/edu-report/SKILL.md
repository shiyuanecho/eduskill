# /edu-report 进展记录

## 做法

1. 用户给一段观察（今天 vs 上周）。
2. 结构化记录：日期 / 任务 / 表现 / 变化。
3. 提示下一步可回到 /edu-diagnosis 或 /edu-lesson。

## 输出格式

- **记录卡**：日期、任务、表现、变化。
- **下一步建议**：回到____（诊断看新卡点 / 游戏练薄弱点）。
- ［非定论标识：观察记录，非测评结论；重大安排请咨询早教/幼教专业人士。］

## 护栏

- 记录的是"变化"，不是"评级"。
- 不把单次表现上升为能力结论。
- 提示回到诊断/游戏，形成闭环而非终点判定。

## 案例（3 正面 + 2 反面）

**正面**
1. 一个 4 岁男孩搭积木，上周要妈妈全程扶着才立得住，这周自己试了三次把塔搭稳了。skill 记下日期/任务/表现/变化——"从需全程帮扶到能自行尝试三次"，并提示回 /edu-lesson 练一练更高塔的平衡，闭环到游戏，没给孩子贴任何标签。
2. 一个 5 岁女孩画画，上周画纸上一团乱线说不清，这周边画边说"这是我的家，外面有太阳"。skill 记录表征与语言的变化，提示回 /edu-diagnosis 看看语言和表征维度有没有新卡点，让记录变成下一步的起点。
3. 一位幼儿园老师记中班一个孩子收玩具：上周要老师点名提醒，这周听到收玩具的音乐自己就开始归位。skill 只记"听到信号后从被动到主动"的变化，并建议回 /edu-lesson 用"送玩具回家"的游戏巩固，没把它当成终点判定。

**反面**
1. 一位家长记"今天娃自己穿好鞋了，说明他自理能力已经很强了"。这走偏了：把一次表现直接上升成能力结论。记录该记的是"和上周比有什么变化"，且能力需要多次、不同情境的观察，单次不能定性。
2. 一位妈妈每天给孩子记一笔，却顺手打星："今天表现 A+，比昨天棒"。这走偏了：把进展记录变成了评级打分，而且记完就结束，没有回到诊断或游戏形成闭环——记录成了终点，而不是帮孩子往前走的一步。
