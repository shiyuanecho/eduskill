# /edu-archive 成长档案

edu-report 记单次；这里把一个孩子的历史攒成一份能长期回看的档案文件。

## 文件约定

- 存在当前工作目录下的 `edu-档案/` 文件夹，一个孩子一个文件，命名 `<孩子代号>.md`。
- 代号用昵称或化名，不用真实全名。文件夹保持扁平，不建子目录。
- 每次更新为**追加**，不覆盖历史。先读旧档案再往下加。

## 做法

1. 用户给孩子代号和年龄。先看 `edu-档案/<代号>.md` 在不在。
   - 不在：新建，写好基础信息 + 本次内容。
   - 在：先读出来，看历史，再在末尾追加本次。
2. 每次追加一条时间戳记录：日期 / 这次关注啥 / 观察到的表现 / 用了什么方案 / 跟上次比有没有变化。
3. 隔几次帮用户回看一下：这段时间哪些在往前走、哪些还卡着、下一步往哪使劲。
4. 提示可回到 `/edu-diagnosis`（看新卡点）或 `/edu-play`（练薄弱点）。

## 档案文件结构（模板）

```
# <代号> 成长档案（<年龄/生日>）

## 基础信息
- 代号 / 大致年龄 / 场景（在家 or 在园）/ 家长或老师关注的重点

## 记录（按时间倒序或正序，追加）
### YYYY-MM-DD
- 关注：____
- 观察：____
- 方案：____
- 变化：跟上次比____

## 阶段回看（每隔几次小结一次）
- 在往前走的：____
- 还卡着的：____
- 下一步：____
```

## 输出格式

- 告诉用户档案存在哪、这次加了什么。
- 需要时给一段阶段回看。
- ［非定论标识：观察记录，非测评结论；重大安排请咨询早教/幼教专业人士。］

## 护栏

- 档案记的是"变化和过程"，不是给孩子"评级"。
- 不把单次表现上升成能力结论。
- 不存真实全名等敏感个人信息，用代号。
- 追加不覆盖，历史不能丢。

## 案例（3 正面 + 2 反面）

**正面**
1. 一位家长给孩子建档案，代号用"小柚子"而非真名。三个月里每次观察都追加到 `edu-档案/小柚子.md`：从"点数漏最后一块"到"能指着一个说一共几块"。回看时发现数感在稳步往前，卡点从点数挪到了一一对应，历史一点没丢。
2. 一位幼儿园老师给中班一个孩子建代号"豆豆"的档案，历次诊断和方案都往下加、不覆盖。学期末做阶段回看：哪些在往前走、哪些还卡着，据此调整了区角的材料，档案真正串起了诊断到教学的闭环。
3. 一位妈妈坚持每次只追加不重写。半年后回头看，发现孩子曾经"怎么都系不上扣子"的事，现在成了日常。把历次记录摆在一起，焦虑明显少了——档案让她看见的是过程，不是某一天的快慢。

**反面**
1. 一位家长把档案当成绩单，每次更新都加一句"本月评级：中上"。这走偏了：档案记的是变化和过程，不是给孩子评级。一旦开始打分，就容易把某次低落当作"退步"，忘了发展本就有起伏。
2. 一位用户更新时直接覆盖了旧文件，还说"反正旧的没用了"；另一次用了孩子真名"张小明"当文件名。这走偏了：覆盖丢了历史，违背"追加不覆盖"；用真名存了敏感个人信息，违背"用代号不用真名"——两条护栏一起破。
