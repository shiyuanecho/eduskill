# /edu-decision 选课决策

## 做法

1. 用户给候选（如"要不要报珠心算 / 思维课 / 英语启蒙"）与自家情况。
2. 拆每个选项成立的前提、得失、适用边界。
3. 产出决策卡：利弊表 + 适用前提 + 后悔风险点。不做"你该选 X"。

## 输出格式

- **选项A**：前提 / 得失 / 适用边界。
- **选项B**：……
- **决策卡**：你家情况是____，更匹配____，但需注意____（后悔风险点）。
- ［非定论标识：观察记录，非测评结论；重大安排请咨询早教/幼教专业人士。］

## 护栏

- 呈现利弊，不下"你该选 X"的指令。
- 不暗示某个选择"错误"。
- 前提核查依据发展顺序，不凭机构话术。

## 案例（3 正面 + 2 反面）

**正面**
1. 一位妈妈在"思维课"和"珠心算"之间犹豫，5 岁男孩坐不住、爱问为什么。skill 先把两者前提摊开：思维课靠生活经验和好奇驱动，珠心算需要大量机械练习、对低龄未必友好；再给决策卡——你家孩子是"为什么型"，更匹配生活化的思维游戏，但需注意"别指望短期算出结果"的后悔风险点。妈妈看清了，没跟风报珠心算。
2. 一位爸爸挑英语启蒙机构，纠结外教小班还是线上 AI 课。skill 拆前提：外教前提是家里中文输入已经充足、孩子适应过集体；线上前提是家长能陪、且替代不了真人来回对话。决策卡点出后悔风险：若中文对话本来就少，先补家里中文量再报班更稳。爸爸决定先把晚饭聊天时间捡起来。
3. 一位幼儿园老师为班里分类活动选教具，在"现成分类盒"和"自然物（树叶、石子）"间拿不准。skill 摊开：现成盒省事但玩法固定，自然物灵活但需老师自己设计。结合本班孩子已玩过基础分类，老师选了自然物做延伸，没花冤枉钱。

**反面**
1. 一位妈妈问"该不该报幼小衔接班"，skill 直接回"你该报，现在不报就晚了"。这走偏了：下了"你该选 X"的指令，却没摊开前提——孩子当下的握笔和倾听水平、家里能不能陪读、机构具体教什么。决策类本意是帮人看清利弊，不是替人拍板。
2. 一位家长想给娃选兴趣班，skill 说"学钢琴比轮滑有用，轮滑就是玩，浪费时间"。这走偏了：把大人的偏好当标准，暗示某个选择"错误"。选课决策只该呈现每个选择的前提与得失，不该给不同兴趣排高低。
